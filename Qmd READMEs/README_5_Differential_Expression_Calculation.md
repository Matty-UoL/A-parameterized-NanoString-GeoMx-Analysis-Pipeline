# 5 Differential Expression Calculation

`5_Differential_Expression_Calculation.qmd` fits the protocol-supported differential expression models and writes structured outputs for later visualization and review.

## When to Run

Run this notebook after stages `0` through `4` have completed and the normalized expression objects are available under the configured normalization output folder.

## Path Portability

Stage 5 defaults to `project_root: "."`, writes DE contracts under `Outputs/5_DE/results/de/`, and performs a 200-character path audit before model fitting. Existing caches under the legacy `results/differential_expression/` directory are copied into the compact directory when reuse is enabled; legacy files are not moved or deleted.
GeoDiff files are optional. If the GeoDiff-filtered count object or the GeoDiff-normalized object is missing, `5_Differential_Expression_Calculation.qmd` skips the branches that require those specific inputs and continues with available non-GeoDiff methods where possible.

Terminology used by this stage:
- `GeoDiff_spatial_data.RDS` is the **GeoDiff-filtered count object**. It supplies count-like data for `dream_GeoDiffCounts` and `limma_voom_GeoDiff`.
- `GeoDiff_normalised_spatial_data.RDS` contains GeoDiff-normalized `normmat`. It supplies the normalized assay for `limma_trend_GeoDiff`.

## Key YAML Decisions

- `out_stage`: defaults to `5_DE`; keep this synchronized with `6_Differential_Expression_Visualisation.qmd`.
- `de_methods`: choose the methods to fit (`limma-trend`, `limma-voom`, and/or `dream`).
- `contrast_factor`: choose the metadata column that defines DE groups. The default `Annotation` is generated as `<cell_type>_<disease_status>`, but any existing metadata column can be used. Separate `cell_type` and `disease_status` columns are only needed for this default `Annotation` factor; if you set `contrast_factor` to any other column (for example a single combined condition or segment column), that column defines the groups on its own and the other role columns fall back to placeholders where absent. Your dataset does not need distinct cell-type and segment columns to use this notebook.
- `model_mode` and `contrast_file`: prefer `pairwise_mixed` with an explicit contrast table for planned biological comparisons. `one_vs_rest_mixed` can be useful for broad screening or exemplar runs, but by averaging diverse groups into a single baseline it can mask true differences and allow a single outlier group to skew the result. Pairwise contrast files should live in `Data/`, for example `Data/contrasts.txt`.
- `col_slide`, `col_patient`, `col_disease_status`, `col_cell_type`, `col_segment`: preferred metadata mappings. The notebook attempts sensible fallbacks when possible.
- `random_effect_var` and `grouping_id_vars`: define the repeated-measures grouping used by supported mixed-model or duplicate-correlation branches.
- `model_covariates_continuous`: numeric fixed-effect adjustment variables, for example `value: ["age"]`. Use this only for measured continuous variables that should enter the model as one numeric term.
- `model_covariates_categorical`: factor fixed-effect adjustment variables, for example `value: ["sex", "slide_name"]`. Use this for group-like technical or biological covariates.
- `q3_norm_assay`, `geodiff_norm_assay`, and `limma_trend_prior_count`: define which normalized assays enter limma-trend and how Q3 is log-transformed.

For pairwise contrasts, the contrast table must contain `factor`, `numerator`, and `denominator`. The `factor` value must match `contrast_factor`; for example, use `Annotation` with `Acinar_Tumor` versus `Acinar_Healthy`, or use a custom factor such as `disease_celltype` with the matching metadata levels.

Treat one-vs-rest results as screening evidence unless the study design explicitly justifies that baseline. For final interpretation and pathway enrichment, define the biologically meaningful numerator/denominator comparisons in a `pairwise_mixed` contrast file and rerun Stage 5.

## Covariate Type Checks

Stage 5 treats continuous and categorical covariates differently by design. A continuous covariate such as age should be numeric before this notebook runs, so it contributes one numeric model coefficient. A categorical covariate such as sex, slide, or batch is converted to a factor and contributes factor-expanded model terms.

Before running Stage 5, inspect the metadata generated by the earlier stages and confirm that numeric covariates are genuinely numeric. If an age column contains values such as `45 years`, `unknown`, or a mixture of text and numbers, correct the source metadata and rerun the staged workflow. This keeps the modeling input reproducible and avoids accidental factor modeling of a continuous variable.

The same typed fixed-effect covariate specification is used by `limma-trend`, `limma-voom`, and `dream`.

When GeoDiff outputs are absent:
- Available without GeoDiff: `limma_trend_Q3` and `limma_voom_TMM`.
- Skipped without `GeoDiff_spatial_data.RDS`: `dream_GeoDiffCounts` and `limma_voom_GeoDiff`.
- Skipped without `GeoDiff_normalised_spatial_data.RDS`: `limma_trend_GeoDiff`.
- Check `method_skip_summary.csv` when GeoDiff files were intentionally absent.

## Outputs to Check

- Method result RDS files and CSV summaries under `Outputs/5_DE/results/de/`.
- Contrast lists and method run-health tables.
- `method_skip_summary.csv`, when optional GeoDiff-required branches were skipped.
- `de_model_covariate_encoding_audit.csv`, which records requested covariate type, imported class, model class, numeric parse failures, and design-matrix columns for each active branch.
- `de_model_input_audit.csv`, which records the exact assay, transformation, dimensions, integer-like status, and missing-value counts used by each DE branch.
- Per-contrast Excel workbooks under `Outputs/5_DE/results/de/`: `<method>_results.xlsx` (every tested gene) and `<method>_results_significant.xlsx` (only genes passing the `adj_p_cutoff`/`logFC_cutoff` thresholds). Each workbook has one worksheet per contrast plus an `_index` sheet mapping the worksheet name to the full contrast name, so results can be browsed or shared without importing the RDS into R.
- Mean-variance diagnostic plots under `Outputs/5_DE/figures/de/` (`meanvar_<branch>.png` for each fitted limma-voom, limma-trend, and dream branch), shown in the notebook's "Mean-variance diagnostics" section. These confirm the mean-variance trend each method assumes and help flag unexpected behaviour such as a strong low-count hook.
- DE-group comparison table under `Outputs/5_DE/tables/`, which reports the groups built from `contrast_factor` across input objects.
- Note: Stage 5 calculates results only. Interpretation figures (volcano plots, heatmaps, gene-level boxplots) are produced in `6_Differential_Expression_Visualisation.qmd`; the `Outputs/5_DE/figures/de/` folder now holds the mean-variance diagnostic plots described above.
- `Outputs/5_DE/tables/output_list.csv`, which records the files produced by the render.
